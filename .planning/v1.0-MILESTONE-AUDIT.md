## Cross-Phase Integration

### Integration Issues

#### INT-01 (HIGH): Per-node config not forwarded to chat backend

- **Affected:** SERV-01, CONF-01
- **From:** `FlowService.updateNodeConfig` -> `FlowService.saveFlow` -> `POST /api/flow`
- **Missing:** `ChatService.sendMessage` sends `{ sessionId, message }` only — never includes `flowConfig`
- **Impact:** User can edit systemPrompt/temperature via CONF-01 panel and persist via CONF-02, but the config is ignored during chat. `chat.post.ts` uses a hardcoded prompt assembled from vector search results.
- **Note:** Both SERV-01 and CONF-01 requirements are still individually satisfied (the service connects, the panel edits). The gap is in the cross-phase data flow.

#### INT-02 (MEDIUM): Nueva Conversacion architectural mismatch

- **Affected:** CONF-05
- **Location:** Button in `ChatComponent` header (Phase 4), not `FlowToolbarComponent` (Phase 5)
- **Impact:** Functionally complete — users can start new conversations. Architecturally, `FlowToolbar` has no `ChatService` injection.

#### INT-03 (MEDIUM): Session restore for expired sessions

- **Affected:** CHAT-04
- **Issue:** `GET /api/sessions/:id` returns 200+empty for unknown sessions instead of 404
- **Impact:** Stale `chat_session_id` in localStorage is never evicted after MongoDB TTL expiry. Not a demo blocker (first-time users start fresh), but breaks session continuity after long gaps.

### Orphaned Routes

3 Phase 3 API routes have no frontend callers:
- `GET /api/vehicles` — consumed server-side via `vectorSearchService` instead
- `GET /api/dates` — no frontend date display feature in v1
- `GET /api/faq` — consumed server-side via `vectorSearchService` instead

These are infrastructure for future direct data access or Persona B integration. Not blocking.
