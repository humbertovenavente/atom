1. **Create `src/server/sse/emitter.ts`** — a typed SSE emission factory function:
   ```typescript
   import type { ServerResponse } from 'node:http';

   export type SSEEventType =
     | 'agent_active'
     | 'message_chunk'
     | 'validation_update'
     | 'done'
     | 'error';

   export type SSEEmitter = (eventName: SSEEventType, data: unknown) => void;

   export function createEmitter(res: ServerResponse): SSEEmitter {
     return (eventName, data) => {
       res.write(`event: ${eventName}\ndata: ${JSON.stringify(data)}\n\n`);
     };
   }
   ```
   This is from 01-RESEARCH.md Pattern 4 — use it verbatim.

2. **Create `src/server/routes/api/chat.post.ts`** — the POST /api/chat endpoint with hardcoded mock SSE pipeline:

   Key requirements:
   - Import `defineEventHandler`, `readBody`, `setHeader` from `h3`
   - Import `createEmitter` from the emitter utility
   - Import `ChatRequest` type from shared types
   - Parse request body as `{ sessionId, message }`
   - Set SSE headers SYNCHRONOUSLY before any await (CRITICAL — see 01-RESEARCH.md Pitfall 1):
     - `Content-Type: text/event-stream`
     - `Cache-Control: no-cache`
     - `Connection: keep-alive`
   - Call `event.node.res.flushHeaders()` immediately after setting headers
   - Export `config = { maxDuration: 60 }` at module level for Vercel function duration
