  <action>
Create `src/app/services/chat.service.ts` as a new Angular injectable service (`providedIn: 'root'`).

**Imports:**
- `Injectable, inject, signal, computed` from `@angular/core`
- `createParser` from `eventsource-parser` (v3 API: `createParser({onEvent})` — NOT `createParser(callback)`)
- `FlowService` from `./flow.service`
- Types from `@models/types`: `ChatMessage`, `AgentActiveEvent`, `MessageChunkEvent`

**AGENT_NODE_MAP constant** (maps AgentType to actual node IDs from flow.get.ts):
```typescript
const AGENT_NODE_MAP: Record<string, string> = {
  memory: 'memory-1',
  orchestrator: 'orchestrator-1',
  validator: 'validator-1',
  specialist: 'specialist-faqs',
  generic: 'generic-1',
};
```

**State signals:**
- `readonly messages = signal<ChatMessage[]>([]);`
- `readonly isStreaming = signal(false);`
- `readonly sessionId = signal<string | null>(null);`

**sendMessage(userText: string): Promise<void>:**
1. Guard: if `isStreaming()` is true, return immediately (prevent double-send)
2. Add user message to messages signal: `{ role: 'user', content: userText, timestamp: new Date() }`
3. Set `isStreaming(true)`
4. Auto-create session if `sessionId()` is null: `fetch('/api/sessions', { method: 'POST' })` → parse JSON → store sessionId in signal AND `localStorage.setItem('chat_session_id', sessionId)`
5. Add empty placeholder assistant message: `{ role: 'assistant', content: '', timestamp: new Date(), agentType: undefined }`
6. POST to `/api/chat` with `{ sessionId, message: userText }` (Content-Type: application/json)
7. Get `response.body.getReader()`, create `TextDecoder`
8. Create parser: `createParser({ onEvent: (event) => this.handleSSEEvent(event) })`
9. Read loop: `while (true) { const { done, value } = await reader.read(); if (done) break; parser.feed(decoder.decode(value, { stream: true })); }`
10. After loop: `isStreaming.set(false)`, `flowService.setActiveNode(null)`, `flowService.clearCompletedNodes()`

**handleSSEEvent(event: { event?: string; data: string }): void:**
- Parse `event.data` as JSON
- Switch on `event.event`:
  - `'agent_active'`: Cast data as AgentActiveEvent. Look up nodeId via `AGENT_NODE_MAP[data.node]`. If status is `'processing'`, call `flowService.setActiveNode(nodeId)`. If status is `'complete'`, call `flowService.setCompletedNode(nodeId)` then `flowService.setActiveNode(null)`.
  - `'message_chunk'`: Cast as MessageChunkEvent. Update messages signal — find the last assistant message and APPEND `data.content` to its content (do NOT replace). Use `messages.update()` to create new array reference.
  - `'done'`: No-op (stream will end naturally)
  - `'error'`: If last message is assistant with empty content, set its content to `data.message ?? 'Error al procesar tu mensaje'`

**loadSession(sessionId: string): Promise<void>:**
1. `fetch(\`/api/sessions/${sessionId}\`)`
2. If `!res.ok` → `localStorage.removeItem('chat_session_id')` and return
3. Parse JSON → set `this.sessionId(sessionId)` → map `conversation.messages` to `ChatMessage[]` format (role, content, timestamp as new Date(), agentType)

**startNewSession(): void:**
1. `this.messages.set([])`
2. `this.sessionId.set(null)`
3. `localStorage.removeItem('chat_session_id')`

**CRITICAL NOTES:**
- Use raw `fetch()`, NOT Angular HttpClient — HttpClient does not expose the ReadableStream body needed for SSE parsing
- eventsource-parser v3 requires `createParser({onEvent})` object syntax — passing a function directly throws `TypeError: callbacks must be an object`
- Always APPEND message_chunk content, never replace — handles both single-chunk mock and future multi-chunk AI responses
- Wrap the fetch + stream loop in try/catch — on error, set isStreaming(false) and show error in last assistant message
