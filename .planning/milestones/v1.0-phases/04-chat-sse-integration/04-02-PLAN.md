**Component setup:**
- Standalone component, selector: `app-chat`
- Imports: `FormsModule` (for ngModel), `NgFor` (for message loop — or use @for), `NgClass`
- Inject: `ChatService`, `DomSanitizer`

**State:**
- `inputText = ''` — bound to input via ngModel
- `userScrolledUp = false` — tracks if user has scrolled away from bottom
- `@ViewChild('messagesContainer') messagesContainer!: ElementRef<HTMLDivElement>` — scroll container ref
- `suggestionChips = ['Ver catálogo de autos', 'Agendar una cita', 'Preguntas frecuentes', 'Opciones de financiamiento']`

**Agent badge config** (for mapping agentType to display label and color):
```typescript
const AGENT_BADGE_MAP: Record<string, { label: string; color: string }> = {
  memory: { label: 'Memoria', color: '#8B5CF6' },
  orchestrator: { label: 'Orquestador', color: '#3B82F6' },
  validator: { label: 'Validador', color: '#10B981' },
  specialist: { label: 'Especialista', color: '#F59E0B' },
  generic: { label: 'Genérico', color: '#6B7280' },
};
```

**Constructor / Init:**
- In constructor, create an `effect()` that watches `this.chat.messages()` — when messages change and `!this.userScrolledUp`, call `setTimeout(() => this.scrollToBottom(), 0)` to scroll after DOM update.
- In constructor (or ngOnInit), use `afterNextRender(() => { ... })` to check localStorage:
  ```typescript
  const savedId = localStorage.getItem('chat_session_id');
  if (savedId) {
    this.chat.loadSession(savedId).catch(() => localStorage.removeItem('chat_session_id'));
  }
  ```

**Methods:**
- `send()`: If `inputText.trim()` is empty or `chat.isStreaming()`, return. Call `chat.sendMessage(inputText.trim())`. Clear `inputText = ''`.
- `onKeydown(event: KeyboardEvent)`: If Enter and not Shift, prevent default and call `send()`.
- `fillInput(chip: string)`: Set `inputText = chip`, then call `send()`.
- `renderMarkdown(content: string): SafeHtml`: `const html = marked.parse(content) as string; return this.sanitizer.bypassSecurityTrustHtml(html);`
- `onScroll(event: Event)`: Check scroll position — `const el = event.target as HTMLDivElement; const atBottom = el.scrollHeight - el.scrollTop - el.clientHeight < 50; this.userScrolledUp = !atBottom;`
- `scrollToBottom()`: `const el = this.messagesContainer?.nativeElement; if (el) el.scrollTop = el.scrollHeight;`
- `isLastMessage(msg: ChatMessage): boolean`: `const msgs = this.chat.messages(); return msgs[msgs.length - 1] === msg;`
- `getAgentBadge(agentType?: string): { label: string; color: string } | null`: Look up in AGENT_BADGE_MAP, return null if not found.

              {{ chip }}
            </button>
          }
        </div>
      </div>

          [class]="message.role === 'user'
            ? 'bg-blue-600 text-white rounded-br-sm'
            : 'bg-gray-800 text-gray-100 rounded-bl-sm'">

          <!-- Agent badge for assistant messages -->
          @if (message.role === 'assistant' && getAgentBadge(message.agentType)) {
            <div class="flex items-center gap-1.5 mb-1">
              <span class="w-2 h-2 rounded-full inline-block"
                [style.backgroundColor]="getAgentBadge(message.agentType)!.color"></span>

          <!-- Message content -->
          @if (message.role === 'assistant' && message.content) {
            <div class="chat-markdown text-sm" [innerHTML]="renderMarkdown(message.content)"></div>

              <span></span><span></span><span></span>

**CRITICAL NOTES:**
- Use `@for` control flow (Angular 17+), NOT `*ngFor` — project uses modern Angular 21
- BUT if imports still need `NgFor`, include it as fallback. Actually, with `@for` you don't need NgFor import.
- `FormsModule` is required for `[(ngModel)]` — add to standalone component imports
- `marked.parse(content) as string` — cast required since marked v15 returns `string | Promise<string>`
- Do NOT call `renderMarkdown()` on every token during streaming — only on assistant messages that have content. The `[innerHTML]` binding with `renderMarkdown()` will re-render on each signal change which is acceptable since marked.parse is fast.
- The `afterNextRender` is used for session restore to ensure we're in browser context (SSR safe — even though SSR is disabled)
