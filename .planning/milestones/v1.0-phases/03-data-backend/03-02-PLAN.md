Create `src/server/services/vector-search.service.ts` following the memoryService pattern (service object with async methods):

1. **Imports:**
   - `OpenAI` from 'openai'
   - `connectDB` from '../db/connect'
   - `Vehicle` from '../models/vehicle'
   - `FAQ` from '../models/faq'

2. **OpenAI client (lazy init):**
   - `let openaiClient: OpenAI | null = null;`
   - `function getOpenAI(): OpenAI` — lazily creates OpenAI client using `process.env.LLM_API_KEY` and optional `process.env.LLM_BASE_URL`. Throws if LLM_API_KEY missing.

3. **Private embed helper:**
   - `async function embedQuery(text: string): Promise<number[]>` — calls `openai.embeddings.create({ model: 'text-embedding-3-small', input: text })`, returns the 1536-dim vector

4. **Exported service object:**
   ```typescript
   export const vectorSearchService = {
     async searchVehicles(query: string, limit = 5): Promise<any[]>,
     async searchFAQs(query: string, limit = 5): Promise<any[]>,
   };
   ```

5. **searchVehicles implementation:**
   - `await connectDB()`
   - Generate query embedding via `embedQuery(query)`
   - Run `$vectorSearch` aggregation:
     ```typescript
     Vehicle.aggregate([
       { $vectorSearch: {
           index: 'vehicles_vector_index',
           path: 'embedding',
           queryVector,
           numCandidates: limit * 10,
           limit,
       }},
       { $project: {
           embedding: 0,
           score: { $meta: 'vectorSearchScore' },
       }},
     ])
     ```
   - Return results array (each doc includes `score` field for relevance ranking)

6. **searchFAQs implementation:**
   - Same pattern but uses FAQ model and `faqs_vector_index`

DESIGN DECISIONS (Claude's discretion):
- No standalone /api/search endpoint — vector search is internal-only, consumed by Phase 4 chat specialists. No demo value in exposing it as HTTP.
- Lazy OpenAI client initialization — the service is imported by route handlers; the OpenAI client should not be created at import time.
- Keep loader.ts unchanged as static fallback — chat.post.ts still imports from it. Phase 4 will wire chat.post.ts to use vectorSearchService instead.
