<task type="auto">
  <name>Task 2: Create seed script with embeddings and vector index creation</name>
  <files>scripts/seed.ts</files>
  <action>
Create `scripts/seed.ts` — a standalone script that:

1. **Environment setup:**
   - `import 'dotenv/config'` (or manually load .env if dotenv not available — use `process.loadEnvFile('.env')` which is built-in Node 20+)
   - Fail fast with clear error if LLM_API_KEY is missing: `if (!process.env.LLM_API_KEY) throw new Error('LLM_API_KEY not set in .env')`
   - Initialize OpenAI client: `new OpenAI({ apiKey: process.env.LLM_API_KEY, baseURL: process.env.LLM_BASE_URL || undefined })` — honor LLM_BASE_URL if present (proxy support)

2. **Embedding helper functions:**
   - `embedBatch(texts: string[], batchSize = 20): Promise<number[][]>` — calls `openai.embeddings.create({ model: 'text-embedding-3-small', input: batch })` in chunks of `batchSize`. Add 1-second delay between batches to respect rate limits.
   - For vehicles, concatenate fields for embedding text: `${Marca} ${Modelo} ${Año} ${Segmento} ${Descripción} Precio: ${Precio} Transmisión: ${Transmisión} Combustible: ${Tipo de combustible}`
   - For FAQs, concatenate: `${pregunta} ${respuesta}`

3. **Data loading and transformation:**
   - Read jsons/autos.json: `JSON.parse(readFileSync('jsons/autos.json', 'utf-8'))`. Extract `.available_vehicles` array.
   - Map each vehicle from Spanish field names (with spaces/accents like "Tipo de combustible") to the camelCase schema fields (tipoCombustible). Keep all original data.
   - Read jsons/faq.json: Extract `.faq_agencia_autos` array. FLATTEN the nested structure — each category has a `preguntas` array, produce one flat document per Q&A with fields: { categoria, pregunta, respuesta, faqId: pregunta.id }.
   - Read jsons/dates.json: Direct array of { fecha, slots } — no transformation needed, no embedding.

4. **Seeding flow (drop & reseed — LOCKED DECISION for hackathon simplicity):**
   - Connect to MongoDB via `connectDB()` from `src/server/db/connect.ts`
   - Drop existing collections: `Vehicle.deleteMany({})`, `FAQ.deleteMany({})`, `DateSlot.deleteMany({})`
   - Generate embeddings for vehicles (100 texts) and FAQs (50 texts) in parallel using `embedBatch`
   - Attach embedding arrays to each document
   - Insert: `Vehicle.insertMany(vehicleDocs)`, `FAQ.insertMany(faqDocs)`, `DateSlot.insertMany(dateDocs)`
   - Log counts: "Seeded: 100 vehicles, 50 FAQs, 31 date slots"

5. **Atlas Vector Search index creation:**
   - Use `mongoose.connection.getClient().db(process.env.MONGODB_DB_NAME)` to access the native MongoDB client
   - Check if indexes already exist via `collection.listSearchIndexes().toArray()` — skip if index with target name exists
   - Create two vector search indexes:
     - `vehicles_vector_index` on `vehicles` collection: `{ type: 'vectorSearch', definition: { fields: [{ type: 'vector', path: 'embedding', numDimensions: 1536, similarity: 'cosine' }] } }`
     - `faqs_vector_index` on `faqs` collection: same structure
   - Use `db.command({ createSearchIndexes: collectionName, indexes: [...] })` syntax
   - Poll `listSearchIndexes()` until both indexes reach `status === 'READY'` or 3-minute timeout
   - Log status during polling: "Waiting for index X... status: BUILDING"

6. **Cleanup:**
   - Disconnect Mongoose: `await mongoose.disconnect()`
   - Exit: `process.exit(0)` on success, `process.exit(1)` on error

IMPORTANT ANTI-PATTERNS TO AVOID:
- Do NOT use src/server/data/ files (catalog.json, faqs.json, schedule.json) — they are a DIFFERENT dataset. Use jsons/ exclusively.
- Do NOT add embedding field to DateSlot documents.
- Do NOT exit before vector search indexes reach READY — queries will return 0 results silently.
- Use the OpenAI constructor with apiKey param, NOT the OPENAI_API_KEY env var (our var is named LLM_API_KEY).
