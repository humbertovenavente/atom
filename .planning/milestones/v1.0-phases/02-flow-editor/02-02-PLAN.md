2. Create `src/app/components/canvas/canvas.component.html`:
   ```html
   <f-flow fDraggable>
     <f-canvas>
       <!-- Edges (render before nodes so they appear behind) -->
       <f-connection
         *ngFor="let edge of flowService.edges()"
         [fOutputId]="edge.source + '-out'"
         [fInputId]="edge.target + '-in'"
         fType="segment"
         class="flow-edge"
         [class.edge--active]="flowService.activeNodeId() === edge.source || flowService.activeNodeId() === edge.target">
       </f-connection>

       <!-- Nodes -->
       <div
         *ngFor="let node of flowService.nodes()"
         fNode
         fDragHandle
         [fNodeId]="node.id"
         [fNodePosition]="node.position"
         [fNodeOutput]="node.id + '-out'"
         [fNodeInput]="node.id + '-in'"
         fOutputConnectableSide="right"
         fInputConnectableSide="left"
         class="flow-node"
         [class.node--active]="flowService.activeNodeId() === node.id"
         [style.borderColor]="node.data.color"
         (click)="flowService.setSelectedNode(node.id)">
         <!-- Node body content -->
         <div class="flow-node__content">
           <span class="flow-node__icon">{{ getEmoji(node.data.icon) }}</span>
           <span class="flow-node__label">{{ node.data.label }}</span>
         </div>
       </div>
     </f-canvas>
   </f-flow>
   ```

   IMPORTANT: The exact @foblex/flow directive API may differ from the template above. During implementation:
   - Verify directive names by checking what FFlowModule exports. Look in node_modules/@foblex/flow for the actual directive selectors.
   - The POC from Phase 1 (described in 01-02-SUMMARY) used: `fNode`, `fDragHandle`, `fNodeOutput` (as element/directive), `fNodeInput` (as element/directive), `fOutputId`, `fInputId`, `fOutputConnectableSide`, `fInputConnectableSide`, `fNodePosition`, `f-connection`, `fDraggable`.
   - If the node output/input ports need to be separate child elements (div with fNodeOutput/fNodeInput directives) rather than attributes on the fNode div, restructure accordingly. The research suggests ports are child elements.
   - Use `[fNodePosition]="node.position"` — bind to the signal array item's position object.
