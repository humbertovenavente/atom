# Requirements: AI Agent Builder Frontend

**Defined:** 2026-02-28
**Core Value:** El editor visual muestra nodos iluminándose en tiempo real mientras el usuario chatea — eso es lo que gana la demo.

## v1 Requirements

### Setup & Foundation

- [x] **SETUP-01**: Proyecto Analog.js creado con Angular 17+, Tailwind CSS, y routing file-based
- [x] **SETUP-02**: Librería de flow editor instalada y funcionando (alternativa a @xyflow/angular)
- [x] **SETUP-03**: Layout principal con 3 paneles: sidebar izq (nodos) + canvas centro (editor) + panel der (chat)
- [x] **SETUP-04**: Interfaces TypeScript compartidas definidas en models/types.ts
- [x] **SETUP-05**: Mock de backend (API routes de Nitro) para desarrollo independiente de Persona B

### Flow Editor

- [x] **FLOW-01**: 6 tipos de nodos custom renderizados en el canvas con colores e iconos distintos (Memoria, Orquestador, Validador, Especialista, Genérico, Tool)
- [x] **FLOW-02**: Nodos son draggables en el canvas (reposicionables con mouse)
- [x] **FLOW-03**: Edges/conexiones entre nodos con animación de flujo
- [x] **FLOW-04**: Flujo default pre-cargado con los 8 nodos conectados correctamente
- [x] **FLOW-05**: Mini-mapa del editor mostrando vista general del canvas
- [x] **FLOW-06**: Controles de zoom (+, -, fit view)
- [x] **FLOW-07**: Sidebar con lista de nodos arrastrables al canvas (drag & drop externo)

### Chat Playground

- [x] **CHAT-01**: UI de chat con campo de texto, botón enviar, y área de mensajes scrolleable
- [x] **CHAT-02**: Burbujas de mensaje diferenciadas para user (derecha) y assistant (izquierda)
- [x] **CHAT-03**: Auto-scroll al último mensaje (respetando cuando el usuario hace scroll up)
- [x] **CHAT-04**: Streaming de respuestas via SSE mostrando texto progresivamente
- [x] **CHAT-05**: Typing indicator (tres puntos animados) mientras el bot procesa
- [x] **CHAT-06**: Nodos del flow se iluminan/resaltan en tiempo real según el agente activo durante el chat

### Configuration

- [x] **CONF-01**: Panel de configuración: click en nodo muestra sidebar con opciones editables (system prompt, temperatura)
- [x] **CONF-02**: Guardar configuración del flow al backend (POST /api/flow)
- [x] **CONF-03**: Cargar configuración del flow desde backend (GET /api/flow)
- [x] **CONF-04**: Botón "Reset Flow" para volver al flujo default
- [x] **CONF-05**: Botón "Nueva Conversación" para limpiar el chat e iniciar sesión nueva

### Services

- [x] **SERV-01**: ChatService con conexión HTTP POST al backend y parsing de SSE events
- [x] **SERV-02**: FlowService con estado del editor (nodos, edges, nodo activo, nodo seleccionado) usando signals
- [x] **SERV-03**: Comunicación ChatService → FlowService para highlight de nodos activos

## v2 Requirements

### Polish

- **POL-01**: Dark mode o tema profesional
- **POL-02**: Tooltips en nodos al hacer hover
- **POL-03**: Responsive design completo para distintas resoluciones
- **POL-04**: Animaciones de transición entre estados

### Advanced

- **ADV-01**: Undo/redo en el editor
- **ADV-02**: Exportar/importar flujo como JSON
- **ADV-03**: Markdown rendering en respuestas del chat

## Out of Scope

| Feature | Reason |
|---------|--------|
| Backend/agentes IA | Persona B lo construye en paralelo |
| Deploy a Vercel | Se hace al final con ambos integrados |
| Tests unitarios | Hackathon de 18h, priorizar funcionalidad |
| Autenticación de usuarios | No es parte del MVP del hackathon |
| Voice input | Alto costo de implementación, zero ROI en demo |
| Undo/redo | Complejidad alta para poco impacto en demo |
